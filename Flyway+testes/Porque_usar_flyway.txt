	Normalmente se utiliza o ORM Hibernate para gerar o schema do banco de dados, então a cada alteração nas entidades é refletida automaticamente no banco de dados, para testes ele é perfeito mas para ambiente de produção é arriscado deixar todo o banco de dados como responsabilidade do ORM Hibernate, na própria documentação dele é indicado administrar os schemas por meio de um script de migração. 
	Pensando por esse lado também não é interessante usar uma ferramenta de administração de banco de dados mas, isso geraria outros problemas como o versionamento e saber qual foi executado ou não.
	Para isso existe o Flyway, nele é possível ter o controle de versões e ter a segurança para atualizar os schemas sem medo de gerar maiores problemas. Ele tem um histórico dentro de uma tabela criada, com isso ele consegue gerenciar esse histórico e fazer as atualizações de forma automática e segura. 
	Olhando para as vantagens do Flyway em relação a outras ferramentas que fazem o mesmo, ele é uma ferramenta feita para o ecossistema Java, ou seja, é uma facilidade gigante para integrar ele com o Spring, para adicionar ele em um projeto é literalmente adicionar ele no pom.xml e já usar, sem precisar configurar nada alarmante.
